# splg_fusion 卡尔曼融合机制说明 (Continuous Filter)

本文档主要说明 `splg_fusion` 中的核心多传感器融合逻辑，即连续滤波器 (Continuous Filter) 是如何将视觉匹配定位结果与高频 IMU 数据进行融合的。

## 1. 核心思想：严格时序与状态回放 (Rollback & Replay)
无人机视觉定位有一个致命弱点：**计算延迟**。当一帧图像的视觉匹配结果出来时，飞机可能已经往前飞了零点几秒甚至一秒。如果直接把这个“旧”的视觉位置当作飞机“现在”的位置，会导致轨迹频繁往后跳变。

为此，本系统采用了一个带历史快照队列的连续卡尔曼滤波器：
1. **保存快照**：系统会不断把高频算出来的每一刻状态（包含时间、位置、速度、协方差等）存入一个历史队列 (`continuous_filter_history_`)。
2. **状态回滚**：当接收到一个图像时间戳为 $t_{visual}$ 的视觉匹配结果时，系统会在历史队列里往前找，把滤波器的状态**回滚（Restore）** 到 $t_{visual}$ 那个时刻。
3. **视觉更新 (Update)**：在 $t_{visual}$ 这个“过去”的时刻，将视觉位置与当时 IMU 积分推算的位置进行卡尔曼更新，计算卡尔曼增益并修正位置和协方差。
4. **IMU重放 (Replay)**：更新完过去的状态后，系统会把 $t_{visual}$ 到当前真实时间 $t_{now}$ 之间的所有 IMU 数据**重新积分（Predict）** 一遍，从而得到一个既吸收了视觉矫正，又没有延迟的“当前”最优状态。

*(核心代码见: `src/fusion/online_ypr_crop_node_fusion.cpp` 中的 `updateContinuousFusionFromVisualLocked`)*

## 2. 预测阶段 (Predict): IMU 预积分
在没有视觉结果到来的时间里，系统依靠高频 IMU 进行航迹推算：
- 利用 `OnlineImuRollPitchTracker` 提供的姿态与加速度进行短时预积分（Preintegration）。
- 根据积分时长 `dt`，将 IMU 计算出的位置增量 (`delta_enu`) 和速度增量 (`delta_vel_enu`) 累加到上一时刻的状态上。
- **协方差膨胀**：预测阶段会根据配置的过程噪声（如 `kf_process_xy_rw_sigma_m_sqrt_s`）以及 IMU 加速度噪声（`imu_preintegration_accel_sigma_mps2`）来增大状态的不确定度（协方差 `cov_enu` 变大）。时间越长，系统对当前纯惯导推算位置的信心就越低。

## 3. 更新阶段 (Update): 视觉观测与门控
当视觉匹配给出绝对地理坐标时：
1. **计算新息 (Innovation)**：即视觉观测位置减去滤波器预测位置。
2. **计算 NIS (归一化新息平方)**：衡量视觉观测偏离预测轨道的程度。
3. **卡方门控 (Chi-Square Gating)**：如果 NIS 超过了设定的阈值 (`kf_visual_nis_gate`，默认为 16.266)，系统会认为这个视觉匹配是一个**严重错误（Outlier 野值）**。
   - **防拉飞保护**：当前系统默认开启了 `kf_visual_reject_on_gate: 1`。这意味着一旦 NIS 超限，系统会**直接拒绝**这个视觉结果，防止飞机轨迹被错误匹配强行拉飞。
4. **自适应协方差缩放**：如果匹配质量变差，系统会动态调大视觉量测的协方差（`kf_visual_adaptive_enable`），降低滤波器对这次视觉结果的信任权重。
5. **卡尔曼增益 (Kalman Gain) 与状态修正**：如果通过了门控，系统会算出增益 $K$，按比例修正位置状态，并**缩小协方差**（代表融合后系统对位置的信心增加了）。

## 4. 容错机制：大 Gap 重启
如果在纯惯导预测期间，IMU 数据因为硬件原因出现了严重的断流（时间间隔 $dt$ 大于 `imu_preintegration_max_gap_sec`，比如卡顿了 2 秒）：
- 传统的积分器会因为巨大的 $dt$ 导致速度和位置直接发散到无穷大（爆炸）。
- 本系统的连续滤波器会**拦截这种发散**，触发 `imu_gap_restart`：
  - 强行将当前速度重置为 0。
  - 将位置协方差强制加上一个巨大的惩罚值（代表“我不知道我在这几秒内飘到哪里去了”）。
  - 等待下一个视觉结果到来时，触发 `visual_reinit_after_gap` 进行强行重定位。

## 5. 数据来源与输出
- **状态维护坐标系**：整个滤波器内部在东北天 (ENU) 局部直角坐标系下进行运算。
- **输出文件**：
  - 所有的融合预测和更新状态都会被高频（节流至 50ms 一次）输出到 `continuous_fusion.csv`。
  - 这个输出包含了每一次动作的原因，例如：
    - `imu_predict`: 正常的 IMU 向前推算。
    - `visual_update_replayed`: 回滚并成功吸收了延迟视觉结果。
    - `visual_reject_nis_replayed`: 回滚后发现视觉结果是野值并拒绝。
    - `imu_gap_restart`: 发生了 IMU 数据大断流保护。
