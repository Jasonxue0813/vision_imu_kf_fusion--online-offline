# splg_fusion 动态裁图机制说明

本文档主要说明 `splg_fusion` 中卫星底图的动态裁剪逻辑：裁图中心是如何选择的、物理裁剪范围是如何计算的、以及相关的配置参数。

## 1. 为什么需要动态裁图？
无人机视觉定位需要将当前相机拍摄的图像与参考卫星地图进行特征匹配。直接在完整的巨大卫星图上搜索不仅计算量极大，而且容易产生大量错误匹配。
因此，我们需要在进行匹配前，围绕飞机的**当前大概位置**裁剪出一小块卫星子图（ROI），缩小搜索空间。

## 2. 裁图中心的选取策略
裁图的核心问题在于：**“以哪个坐标作为中心来截取卫星图？”**

为了保证在无 GNSS（或 GNSS 漂移）、以及起步阶段的不同场景下都能顺利工作，系统采用了多级回退策略（Fallback Strategy）。在每次需要对相机图像进行视觉匹配前，裁图中心 `center_hit` 的确定优先级如下：

### 优先级 1：上一次的成功视觉融合结果
- **前提条件**：系统已经至少成功进行过一次视觉匹配，并且**上一次成功融合定位的时间距离当前不超过 1 秒**。
- **说明**：只要系统在正常工作，我们总是相信“上一次基于视觉矫正后的位置”是最准的。用它作为下一帧的裁图中心，能够保证裁出来的地图始终跟随飞机的真实运动，不会因为 GNSS 的漂移而被带偏。

### 优先级 2：从最近的连续融合状态出发，利用 IMU 传播到当前图像时刻的预测位置
- **前提条件**：如果由于图像质量差或匹配失败，导致连续超过 1 秒没有成功更新过视觉结果，系统就无法使用优先级 1。
- **说明**：此时系统会回退，去查询**最近一次的连续滤波器（Continuous Filter）历史状态**，然后利用 IMU 数据将其传播积分至当前的图像时刻。虽然视觉跟丢了，但系统一直在靠 IMU 和最后一次已知的速度往前推算。利用这个积分位置裁图，能最大程度保证即便短暂盲飞，裁出来的地图也依然大概率覆盖飞机的当前位置。

### 优先级 3：当前帧时间对应的原始 GNSS 坐标
- **前提条件**：如果连历史积分位置都找不到（比如系统刚起步的第一帧，什么历史缓存都没有）。
- **说明**：最底层的兜底策略，直接使用与当前图像时间戳对齐的 GNSS 经纬度。

*(对应源码位置: `src/vision/online_ypr_crop_node_vision.cpp` 中的 `tryGetLatestFusionCropSeedLocked` 等逻辑)*

## 3. 物理裁剪范围与偏移参数

确定了裁图中心后，我们需要以它为基准圈出一个矩形。这一步由以下几个配置参数控制（定义在 `config/fusion_config.yaml` 的 `tuning` 下）：

### 3.1 裁图大小 (Extent)
系统的裁图大小采用了**动态自适应与固定兜底相结合**的机制：

1. **动态自适应裁剪 (主要方式)**
   - 系统会根据配置的相机对角线视场角 (`crop_dfov_deg`) 和当前图像宽高比，计算出水平 (HFOV) 和垂直视场角 (VFOV)。
   - 结合图像时刻对应的真实海拔 (`image_altitude_m`) 与配置的地平海拔 (`ground_altitude`)，计算出相对高度。
   - 利用三角函数计算出当前相机视野在地面上的实际物理覆盖范围（即幅宽），再乘以一个缩放因子 `crop_scale_factor`，得到动态的 `half_e` 和 `half_n`。
   - **说明**：这种方式保证了飞机在 800m 高空和 3000m 高空时，裁出的地图物理范围能够自动适应视野变化，避免低空裁图过大导致性能浪费，或高空裁图过小导致视野越界。

2. **固定兜底裁剪 (Fallback)**
   - `crop_half_extent_e_m`: 自适应裁图不可用时，东西方向（East）上的**最小半边长**（米）。
   - `crop_half_extent_n_m`: 自适应裁图不可用时，南北方向（North）上的**最小半边长**（米）。
   - **说明**：当 GNSS 海拔不可用或 DFOV 配置不合理导致自适应计算失败时，系统会回退使用这两个基础尺寸进行裁剪，确保流程不中断。

### 3.2 裁图中心前视/偏视偏移 (Offset)
相机的光轴并不一定总是垂直朝下的，有时为了提前看到前方的景物，相机会有一定的俯仰安装角。因此裁图中心不一定非要等于飞机的正下方。
- `crop_center_offset_forward_m`: 裁图中心相对于飞机当前位置向**机头正前方**偏移的距离（米）。
- `crop_center_offset_right_m`: 裁图中心相对于飞机当前位置向**机身右侧**偏移的距离（米）。
- **说明**：系统会利用当前估计的**航向角 (Heading)**，将这两个机体系下的偏移距离转换到东北天 (ENU) 坐标系中，并加到前面选定的裁图中心点上。如果这两个值都配置为 `0.0`，则代表裁图正中心就是飞机所在位置。

## 4. 裁图流程总结
1. 获取当前帧对齐的 GNSS、航向、IMU 姿态。
2. 按照 **优先级策略**（上次视觉结果 > 1秒前积分结果 > 当前 GNSS）选定基础中心点。
3. 利用航向角，将 `offset_forward` 和 `offset_right` 施加到中心点上。
4. 以偏移后的中心点为圆心，向东西延伸 `extent_e`，向南北延伸 `extent_n` 算出经纬度包围盒。
5. 从庞大的全局 TIF 地图文件中，切出这个包围盒对应的栅格图像（即最终用于匹配的 ROI 卫星图）。
